# NLType 打字游戏 — 细化界面设计方案

## 一、整体风格定调

延续“简洁明亮、专注书写”的基调，但结构更丰富、层级更清晰。主背景为柔和的米白（#F7F8FA），主卡片为纯白（#FFFFFF），辅以极浅的蓝灰阴影，整体像一张悬浮在空中的书写纸。所有交互元素均使用圆角、细线、轻投影，视觉噪音降到最低。


## 二、顶部导航栏（从左至右）

- **左侧品牌区**：`NLType`，字体为加粗无衬线，字距宽松，颜色深蓝灰（#1E293B），后跟一个极小的圆点分隔符（#CBD5E1）。

- **中央模式切换（核心新增）**：
  - 两个胶囊形按钮并排：“单人模式”与“多人模式”。
  - 当前激活模式为实心品牌色背景（#4F7CFF），白色文字；未激活模式为透明背景，深灰色文字（#64748B），悬停时背景出现极浅灰色。
  - 切换时有平滑的填充颜色过渡动画。

- **右侧功能区（从左至右排列）**：
  1. 导航链接：“游戏”（当前页面）与“排行榜”，纯文字，字距宽松，当前页有下划线或小圆点标记。
  2. 分隔线：一条极细的竖线（#E2E8F0）。
  3. **语言切换**：一个简洁的下拉触发器，显示当前语言缩写（如“EN”）。点击后弹出小型下拉菜单，包含“English”与“中文”等选项，选项带有悬停背景色。
  4. **主题切换**：一个圆形按钮，浅色模式下显示月亮线条图标，深色模式下显示太阳线条图标。点击时伴随平滑旋转动画，同时全局背景与卡片颜色瞬间过渡。
  5. **用户入口**：一个幽灵风格按钮（细边框、透明背景），文字为“登录”或用户头像（若已登录则为圆形头像占位）。


## 三、主游戏卡片（核心视觉中心）

卡片宽度固定为 780px（桌面），水平居中，顶部与导航栏间距 48px，四周拥有柔和阴影（0 8px 30px rgba(0,0,0,0.04)）。

### 1. 模式与设置区（卡片顶部）

- **第一行（单人模式时）**：
  - 标签行：`时间 · 单词 · 引用 · 代码 · 禅`。
  - 每个标签为小字号（13px）、大写字母、字距 0.15em。当前激活标签下方有一条 20px 长的品牌色细线（#4F7CFF），其他标签为浅灰色（#94A3B8），悬停变深灰。

- **第一行（多人模式时）**：
  - 标签行保持不变（仍可选择文本类型），但右侧新增一个“房间信息”区域：
    - 显示房间代码（如 `房间 · A7K3`），字体为等宽、小号、灰色。
    - 旁边有一个复制链接的图标按钮，点击后短暂显示“已复制”提示。
    - 若尚未匹配到对手，显示等待状态文字“正在匹配对手...”，并伴有细小脉冲动画（透明度淡入淡出）。

- **第二行（时长选项）**：
  - 仅单人模式下显示，为数字标签：`15 · 30 · 60 · 120`。
  - 当前选中项为圆角矩形背景（品牌色柔和版 #E8EDFF），文字为品牌色；未选中项为纯文字灰色。
  - 多人模式下此区域隐藏，或统一固定为 30 秒（保持竞技公平）。

### 2. 文本展示区（卡片核心）

- **间距与字号**：上下左右均有充足内边距（32px），文本字号为 26px（桌面），行高 1.8。
- **字体**：使用等宽字体（如 JetBrains Mono、Fira Code），确保字符对齐精准。
- **字符状态**：
  - **未输入字符**：浅灰色（#CBD5E1）。
  - **正确输入字符**：深色（#1E293B），字重正常。
  - **当前待输入字符**：深色且下方有一条细小的品牌色竖线光标（宽度 2px，高度与字符一致），光标有平缓的呼吸闪烁（0.6s 周期）。
  - **错误输入字符**：柔和的珊瑚色（#F87171），并带有极细的波浪下划线（仅装饰，不干扰阅读）。
- **多人模式下的双光标**：
  - 文本块中央垂直分割为上下两行（或同一行内用两种颜色区分）。
  - 方案一（上下分行）：上方为玩家 1 的进度，下方为玩家 2 的进度，两个光标颜色不同（玩家 1 为蓝色 #4F7CFF，玩家 2 为橙色 #F59E0B）。
  - 方案二（混合显示）：同一行文本中，已输入部分按玩家分别着色（蓝色区块与橙色区块交替），光标也分别闪烁。推荐方案一，更清晰易读。

### 3. 进度条（卡片底部）

- 一条极细的水平线（高度 2px），从卡片左边缘延伸到右边缘。
- 填充色为品牌色（#4F7CFF），背景为极浅灰（#F1F5F9）。
- 单人模式下：进度 = 已输入字符数 / 总字符数。
- 多人模式下：进度条可并排显示两条（上下排列），或合并为一条显示“整体完成度”，上方用两个小圆点标记两位玩家的当前位置。


## 四、实时统计栏（卡片正下方）

位于卡片之外，独立成一行，宽度与卡片对齐（780px）。

### 单人模式：
横向四列等宽分布，每个指标包含：
- 数值（大号，加粗，深色，如 `84`）
- 单位/标签（小号，灰色，如 `wpm`、`%`、`chars`、`sec`）

四个指标依次为：
1. 速度（WPM）
2. 准确率（Accuracy %）
3. 进度（如 `42 / 200`）
4. 用时（倒计时或已用时间，格式 `00:24`）

### 多人模式：
统计栏分为左右两个区块，中间有一条极细的分隔线：
- **左区块（你）**：显示你的 WPM 与准确率，数值颜色为蓝色（#4F7CFF）。
- **右区块（对手）**：显示对手的 WPM 与准确率，数值颜色为橙色（#F59E0B）。
- 若某一方领先，其对应区块会有一个微弱的“领先”发光效果（极淡的品牌色光晕）。
- 进度部分在中间或下方单独显示，以 `你 ██████░░░░ 对手 ████░░░░░░` 的微型进度条呈现。


## 五、主操作按钮（统计栏下方）

一个宽大的圆角矩形按钮（高度 56px，宽度 280px，居中）。

- **单人模式**：
  - 空闲状态：背景为极浅灰（#F1F5F9），文字“按任意键开始”，颜色灰色（#94A3B8）。
  - 激活状态：背景变为品牌色（#4F7CFF），文字“输入中...”，颜色白色。
  - 完成状态：短暂变为绿色（#10B981），文字“完成！”。

- **多人模式**：
  - 等待对手：按钮显示“等待对手加入...”，背景为浅橙色（#FEF3C7），文字橙色。
  - 已就绪：按钮显示“准备就绪，点击开始”，背景为品牌色。
  - 比赛中：按钮不可点击，显示“对战进行中...”。
  - 结束：显示“查看结果”，点击后展开结果弹窗。


## 六、底部附加信息（页面最底部）

一行极小字号（11px）的灰色文字（#94A3B8），居中显示当前设置的摘要：
- 单人模式示例：`英文 · 30秒 · 时间模式`
- 多人模式示例：`英文 · 30秒 · 多人竞技 · 房间 A7K3`

右侧不放置额外元素（语言与主题已移入顶部导航栏）。


## 七、深色模式与浅色模式的具体切换效果

切换按钮位于右上角，点击后：

- **浅色模式（默认）**：
  - 页面背景 #F7F8FA
  - 卡片背景 #FFFFFF
  - 主文字 #1E293B
  - 辅助文字 #64748B / #94A3B8
  - 品牌色 #4F7CFF
  - 阴影透明且浅

- **深色模式**：
  - 页面背景 #0F172A（深邃蓝黑）
  - 卡片背景 #1E293B（深灰蓝）
  - 主文字 #E2E8F0（亮灰白）
  - 辅助文字 #94A3B8
  - 品牌色 #60A5FA（稍亮以保持对比度）
  - 阴影几乎不可见，用微弱的发光边框替代（#334155）

切换时，所有颜色变化在 300ms 内平滑过渡，无闪烁或跳跃。


## 八、多人模式的特殊界面状态细化

### 1. 匹配等待状态
- 卡片中央显示一个轻柔的脉动环形（品牌色），下方文字“正在寻找对手...”。
- 文本展示区暂时隐藏或显示占位符“等待对手加入”。

### 2. 比赛进行状态
- 文本展示区正常显示，双光标同时移动。
- 统计栏实时更新双方数据。
- 若一方完成全部输入，其光标变为“已完成”标记（对勾），并等待另一方完成。

### 3. 比赛结束状态
- 卡片中央弹出一个简洁的结果横幅（半透明背景遮罩）：
  - 显示胜者（“你赢了！”或“对手获胜”）。
  - 展示双方最终 WPM 与准确率对比。
  - 提供“再来一局”与“返回大厅”两个按钮。


## 九、交互反馈与动效（补充细化）

- **启动加载**：页面元素按“导航栏 → 卡片（含文本） → 统计栏 → 按钮”的顺序，每次间隔 80ms，从下往上轻微位移（12px）并淡入。
- **按键反馈**：每按一键，光标右侧出现极短暂的微光闪烁（200ms），表示输入被接收。
- **错误反馈**：除字符变色外，卡片右侧会短暂出现一个红色的微小圆点提示（400ms 后消失），不会打断输入节奏。
- **进度更新**：进度条填充动画为缓动函数（ease-out），非匀速，更自然。
- **模式切换**：单人/多人切换时，卡片内容（如统计栏布局、按钮文字）会有淡入淡出过渡（200ms），避免突兀。


## 十、响应式行为（简要）

- **桌面（> 1024px）**：卡片宽度 780px，所有元素完整展示。
- **平板（768px - 1024px）**：卡片宽度自适应 90%，统计栏字号略微缩小，双光标分行显示仍然清晰。
- **手机（< 768px）**：
  - 顶部导航栏折叠：品牌左侧，右侧仅保留主题切换与语言切换（图标形式），用户入口隐藏至汉堡菜单。
  - 统计栏改为两行：第一行显示 WPM 与准确率，第二行显示进度与用时。
  - 文本字号缩小至 20px，卡片内边距减半。
  - 多人模式下，双光标不再分行，改用颜色区分（蓝 / 橙）在同一行中交替显示。


## 十一、代码可扩展性设计

### 1. 类型系统（`src/types/`）

所有共享类型集中定义，禁止在页面组件中内联定义类型。

| 文件 | 内容 |
|------|------|
| `types/game.ts` | `GameMode`, `Language`, `GameConfig`, `ModeEntry`, `MODES` 数组, `TIME_OPTIONS`, `WORD_OPTIONS`, `defaultConfig()` |
| `types/results.ts` | `GameResult`, `LeaderboardEntry` — 游戏结算与排行榜数据结构 |

**新增游戏模式：** 在 `GameMode` 联合类型中添加字面量 → 在 `MODES` 数组中追加条目（含 `enabled: true/false`） → HomePage 自动渲染新标签。选项按钮逻辑通过 `mode === "xxx"` 分支处理。

**新增排行榜字段：** 在 `GameResult` 中追加字段 → `saveResult()` 自动持久化 → `getLeaderboard()` 返回新字段 → LeaderboardPage 渲染。

### 2. 全局状态管理（`src/context/`）

所有跨页面共享的状态通过 React Context 提供：

| Context | 文件 | 管理内容 |
|---------|------|---------|
| ThemeContext | `context/ThemeContext.tsx` | `theme: "light" / "dark"`, `toggle()` — 持久化到 localStorage |
| LanguageContext | `context/LanguageContext.tsx` | `language: "en" / "zh"`, `setLanguage()` — 持久化到 localStorage |

Context Provider 按层级包裹在 `App.tsx` 中：
```
ThemeProvider → LanguageProvider → ConfigProvider → BrowserRouter → Routes
```

**新增全局状态：** 创建 `context/XxxContext.tsx` → 在 `App.tsx` 中添加 Provider → 页面/组件通过 `useXxx()` hook 消费。遵循 "Provider 在顶层，hook 在消费处" 模式。

### 3. 主题系统（CSS Variables）

所有颜色通过 CSS 自定义属性定义，支持双向主题：

```css
:root {               /* 浅色模式默认值 */
  --bg-page: #F7F8FA;
  --text-primary: #1E293B;
  --accent: #4F7CFF;
  /* Arco Design 组件变量覆盖 */
  --color-bg-2: #FFFFFF;
  ...
}
:root[data-theme="dark"] {  /* 深色模式覆盖 */
  --bg-page: #0F172A;
  --text-primary: #E2E8F0;
  --accent: #60A5FA;
  /* Arco Design 组件变量覆盖 */
  --color-bg-2: #1E293B;
  ...
}
```

Tailwind 配置引用 CSS 变量（`tailwind.config.js`）：
```js
colors: {
  body: "var(--bg-page)",
  text: { DEFAULT: "var(--text-primary)", ... },
  accent: { DEFAULT: "var(--accent)", ... },
}
```

**新增主题色：** 添加 CSS 变量 → 在 Tailwind config 中添加映射 → 使用 `text-[var(--xxx)]` 或 Tailwind 类名。

**新增第三主题（如 high-contrast）：** 添加 `:root[data-theme="high-contrast"]` 变量块 → ThemeContext 扩展 `theme` 类型。

### 4. 数据持久化（`src/services/`）

基于 localStorage 的抽象层，所有存储操作通过 `services/storage.ts` 进行：

```ts
// services/storage.ts — 泛型工具
load<T>(key: string, fallback: T): T
save(key: string, value: unknown): void
remove(key: string): void

// services/results.ts — 业务层
saveResult(config, stats): GameResult
getResults(): GameResult[]
getLeaderboard(limit): LeaderboardEntry[]
useResults(): { results, add, saved }
```

**切换到后端 API：** 只需替换 `services/storage.ts` 中的 `load`/`save` 实现为 `fetch()` 调用，上层代码无需改动。

**新增持久化需求：** 在 `services/` 下新建模块文件，复用 `storage.ts` 的 `load`/`save` 工具函数。

### 5. 路由结构

```
/                → HomePage   (配置页面)
/leaderboard     → LeaderboardPage (排行榜)
/game            → GamePage   (游戏页面，通过 location.state 接收配置)
```

新增页面只需：
1. 创建 `pages/NewPage.tsx`
2. 在 `App.tsx` 的 `<Routes>` 中添加 `<Route path="/xxx" element={<NewPage />} />`
3. 在 `NavBar` 中添加导航链接

### 6. 游戏引擎（Hooks）

核心游戏逻辑封装在独立 hooks 中，与 UI 完全解耦：

| Hook | 职责 | 多人模式扩展点 |
|------|------|-------------|
| `useTypingEngine` | 键盘事件监听、字符匹配、WPM 计算 | 需重构为接收 `onKeystroke(senderId, char, position)` 回调，支持远程同步 |
| `useTimer` | 倒计时 / 计时器 | 多人场景需支持服务端时钟同步 |

新增游戏玩法（如限时模式、段落模式）只需替换 `generateText()` 函数和传入 `TypingGame` 的 `timeLimit` 参数，引擎逻辑不变。

### 7. 组件分层

```
pages/          ← 页面级组件，有路由
  HomePage       (#配置) → 组合 ModeSelect + OptionButtons + StartButton
  GamePage        (#游戏) → 组合 TypingGame
  LeaderboardPage (#排行) → 组合 ResultsTable
components/     ← 可复用 UI 组件
  NavBar          (#导航) → 消费 ThemeContext, LanguageContext
  TypingGame      (#游戏壳) → 组合 TypingDisplay + StatsBar + ProgressBar + ResultModal
  TypingDisplay   (#文本展示) → 纯展示，接收 chars[] + curren tIndex
context/        ← React Context
hooks/          ← 自定义 Hooks（引擎、计时器）
services/       ← 数据存取抽象
types/          ← 共享类型定义
data/           ← 静态数据（单词表、中文文本）
```

新功能开发流程：确定影响层级 → 修改 types → 实现 service/hook → 创建/修改 component → 组装到 page → 添加路由。
