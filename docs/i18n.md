# 多语言 UI

UI 文本集中管理在 `src/context/I18nContext.tsx`，通过 `useI18n()` 的 `t(key)` 函数引用。

## 添加新语言

以添加日语 (`ja`) 为例：

### 1. 扩展类型

```ts
// src/context/I18nContext.tsx
export type UILang = "zh" | "en" | "ja";
```

### 2. 添加翻译条目

```ts
export const dict: Record<UILang, Record<string, string>> = {
  zh: { /* 已有 */ },
  en: { /* 已有 */ },
  ja: {
    "nav.game": "ゲーム",
    "nav.leaderboard": "ランキング",
    "home.start": "Enter キーで開始",
    "auth.login": "ログイン",
    // ... 全部 key 都需翻译
  },
};
```

> `dict` 的类型是 `Record<UILang, Record<string, string>>`，漏掉一个 key TypeScript 不会报错。确保新语言的 key 数量和已有语言一致（可通过运行时检查或人工对照）。

### 3. 在 UI 添加切换入口

```tsx
// src/components/NavBar.tsx
{(["zh", "en", "ja"] as const).map((l) => (
  <Menu.Item key={l}>
    {uiLang === l ? "✓ " : ""}
    {l === "zh" ? "中文" : l === "en" ? "EN" : "日本語"}
  </Menu.Item>
))}
```

### 4. 默认语言（可选）

```ts
// I18nProvider 中的默认值，按需修改
const [lang, setLang] = useState<UILang>(() => load<UILang>(UI_LANG_KEY, "zh"));
```

## 关键类型

| 符号 | 类型 | 说明 |
|------|------|------|
| `UILang` | `"zh" \| "en"` (联合类型) | 所有合法语言 ID |
| `dict` | `Record<UILang, Record<string, string>>` | 翻译字典 |
| `t(key, vars?)` | `(string, vars?) => string` | 取值函数，支持 `{var}` 模板 |

## 模板变量

部分翻译包含动态值，如 `"用时 {m}分{s}秒"`。添加新语言时保留 `{变量名}` 即可：

```ts
ja: {
  "game.elapsed": "{m}分{s}秒経過",
}
```
